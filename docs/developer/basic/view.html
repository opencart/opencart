Views

Views are Twig templates. The controller hands the loader a route and a data array; the loader resolves the template path and lets Twig render it.

Where templates live

PathNotes
admin/view/template/<group>/<name>.twigCore admin templates.
catalog/view/template/<group>/<name>.twigCore storefront templates (default theme).
extension/<code>/admin/view/template/...Admin templates shipped by an extension.
extension/<code>/catalog/view/template/...Catalog templates shipped by an extension.

Extension and theme overrides

The catalog template loader resolves a route in this order:

  1. Active theme's override directory (catalog/view/theme/<theme>/template/...)
  2. Patched OCMOD copy (extension/ocmod/...) if the route was rewritten by event/modification.view
  3. Default catalog template

This is why themes can override any storefront page without touching the source.

Calling a template from a controller

```php $data['title'] = 'Hello'; $this->response->setOutput($this->load->view('demo/hello', $data)); ```

The route uses forward slashes and no extension. The loader fires view/<route>/before and view/<route>/after events around the render -- this is the hook OCMOD uses to redirect $route to a patched template.

Passing data

The data array becomes the Twig variable scope. Top-level keys are accessible as variables:

```php $data['user'] = ['name' => 'Ada', 'role' => 'admin']; ``` ```twig {{ user.name }} ({{ user.role }}) ```

For HTML that should not be re-escaped (already-rendered partials), pass it as a string and use it directly:

```twig {{ header }} ```

OpenCart's templates rely on partials being safe HTML, so they do not use |raw; do not introduce |raw on values that come from user input.

Common Twig tags

TagUse
{{ value }}Print, HTML-escaped.
{% if x %}...{% endif %}Conditional.
{% for item in items %}...{% endfor %}Iteration. loop.index, loop.first, loop.last available inside.
{% set x = ... %}Local variable.
{{ value|escape }} / |eExplicit HTML escape.
{{ value|default('-') }}Fallback if null/empty.
{{ value|number_format(2) }}Number formatting.

Including partials

For static partials use {% include 'common/partial.twig' %}. For partials that require their own controller logic, render them as sub-controllers and pass the result as data:

```php $data['header'] = $this->load->controller('common/header'); ``` ```twig {{ header }} ```

Assets

Static files (images, CSS, JS) live under view/image/, view/javascript/, view/stylesheet/ on the same side (admin or catalog). The startup chain prepends each enabled extension's view paths so referencing image/foo.png resolves through that search path.

Use the document API to add stylesheets and scripts so they appear in {{ styles }} and {{ scripts }} blocks of the layout:

```php $this->document->addStyle('view/stylesheet/extension/hello.css'); $this->document->addScript('view/javascript/extension/hello.js'); ```
