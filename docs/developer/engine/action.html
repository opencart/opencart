Action

\Opencart\System\Engine\Action wraps a controller route plus its method into a single executable object. The framework uses it for the request's main action, every pre-action, every event listener, and the error fallback.

Shape of an action

An action stores three private fields:

FieldSourceExample
routeThe full input string, sanitised to [a-zA-Z0-9_|/.]common/dashboard.save
controllerThe route up to the last .common/dashboard
methodThe route after the last ., or "index" if nonesave

Constructing one

```php $action = new \Opencart\System\Engine\Action('catalog/product.save'); ```

The route is sanitised by preg_replace('/[^a-zA-Z0-9_|\/\.]/', '', $route). Anything outside the allowed alphabet is silently dropped, so you should not rely on the constructor as a validator.

Executing it

```php $result = $action->execute($registry, $args); ```

The method:

  1. Refuses to call magic methods (anything starting with __).
  2. Looks up the controller in the registry under fallback_controller_<controller>; if absent, asks the factory to construct it and caches it.
  3. If the resolved object is not a Controller, returns a "could not load controller" exception.
  4. Calls the requested method with the unpacked args and returns its result.

What execute() returns

execute() returns whichever of these comes first:

  • The method's own return value (anything).
  • An Exception instance when the controller could not be loaded or the method does not exist or is magic. Note: it returns the exception, it does not throw it.

The framework's bootstrap loop (system/framework.php) checks for these:

  • A returned Action from a pre-action triggers a redirect (replaces the main action and breaks the loop).
  • A returned Exception swaps in action_error.
  • Anything else (or null) means "continue".

Why it exists

Actions are the abstraction that lets the same dispatch path serve four different needs:

  • The request's main controller (set up at the end of pre-actions, executed by the bootstrap).
  • Pre-action items in action_pre_action.
  • Event listeners registered via $event->register($trigger, new Action(...), $priority).
  • The error fallback (action_error).

Storing them as data instead of executing inline lets the framework reorder, swap or short-circuit them; that is what action_event registration and the controller/*/before rewriting depend on.
