Session

\Opencart\System\Library\Session is the session facade. It owns the session id, exposes a data array for read/write, and forwards storage operations to the configured adaptor.

Adaptors

EngineFileNotes
filesystem/library/session/file.phpDefault. Files under system/storage/session/.
dbsystem/library/session/db.phpStores rows in oc_session; survives load-balanced deployments.
redissystem/library/session/redis.phpRedis-backed.

Pick the adaptor with session_engine, the cookie name with session_name, the lifetime with config_session_expire and the SameSite policy with config_session_samesite.

API

MethodDescription
start(string $session_id = ''): stringResumes the session if a non-empty id is given, otherwise creates a new one. Returns the active id.
getId(): stringThe active id.
close(): voidPersists the data array and releases any locks.
destroy(): voidDeletes the storage record for the active id.
gc(): voidGarbage-collects expired sessions.

The data array

Once start() has run, the session exposes a public $data array used by every part of the framework:

```php $this->session->data['user_token'] = oc_token(32); $this->session->data['cart'][$product_id] = $quantity; if (!empty($this->session->data['success'])) { $message = $this->session->data['success']; unset($this->session->data['success']); } ```

Anything stored under a well-known key shows up across the framework: cart drives the cart library, customer_id drives the customer library, user_token is checked on every admin route.

Cookie hardening

startup/session emits the cookie with secure, httponly and the configured SameSite on every request. Do not call setcookie() for the session cookie yourself -- let the startup own it.
