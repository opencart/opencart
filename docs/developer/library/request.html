Request

\Opencart\System\Library\Request is the input wrapper. At construction time it copies $_GET, $_POST, $_COOKIE, $_FILES and $_SERVER into typed public arrays after passing every value through clean().

Properties

PropertySource
$request->get$_GET
$request->post$_POST
$request->cookie$_COOKIE
$request->files$_FILES
$request->server$_SERVER

What clean() does

Every scalar is run through trim(htmlspecialchars(..., ENT_QUOTES, 'UTF-8')). This is a safety net for templates -- any text echoed back to the page is already HTML-safe -- but it is not a substitute for SQL escaping. Always cast/escape values before putting them in a query.

Keys are cleaned the same way, so attempts to inject HTML through a form field name are neutralised at parse time.

Typed accessors

```php $id = $this->request->get('product_id', 'int'); $page = $this->request->get('page', 'int'); $filter = $this->request->post('filter', 'array'); $status = $this->request->post('status', 'bool'); ```

The second argument is the cast: string, int, float, bool, array. Without it, the raw value is returned (or null if the key is missing).

Common patterns

```php if (($this->request->server['REQUEST_METHOD'] === 'POST') && $this->validate()) { // ... save ... } if (isset($this->request->get['filter_name'])) { $filter_name = (string)$this->request->get['filter_name']; } else { $filter_name = ''; } ```

Direct array access ($this->request->get['x']) is the most common style across the core. The typed accessors are newer; both are valid.

Files

```php if (!empty($this->request->files['upload']['tmp_name']) && is_uploaded_file($this->request->files['upload']['tmp_name'])) { // ... } ```

The framework does not pre-validate uploads. Always check error, tmp_name, MIME type and the file extension before accepting.
