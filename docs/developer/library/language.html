Language

\Opencart\System\Library\Language is the translation store. Language files are PHP files that fill $_['key'] = 'value'; the library merges them into one flat map and exposes get / set / all.

API

MethodDescription
__construct(string $code)The active language code, e.g. en-gb.
load(string $filename, string $prefix = '', string $code = ''): arrayLoads a language file and merges its $_ into the store. Most callers go through $this->load->language(...) instead, which also fires events.
get(string $key): stringReturns the translation, or the key itself if missing (so untranslated strings are visible during development).
set(string $key, string $value): voidOverrides a single string at runtime.
all(string $prefix = ''): arrayReturns the full map, optionally filtered to keys starting with $prefix.
clear(): voidWipes the in-memory store.
addPath(string $namespace, string $directory = '')Registers an extension's language directory so its files resolve under a namespace.

Loading from a controller

```php $this->load->language('common/dashboard'); $title = $this->language->get('heading_title'); $data = array_merge($data, $this->language->all()); ```

The third call is the idiom that lets templates use bare variables ({{ heading_title }}) instead of having to call {{ language('heading_title') }}.

Where files live

PathNotes
admin/language/<code>/<route>.phpCore admin strings.
catalog/language/<code>/<route>.phpCore storefront strings.
extension/<ext>/<side>/language/<code>/...Strings shipped by an extension.
extension/ocmod/<side>/language/<code>/...OCMOD-patched copies (the loader redirects $route here when one exists).

Two-letter prefix

The optional $prefix on load() namespaces the keys. Used by the catalog mail templates so order-confirmation strings do not collide with checkout strings even though both files might define text_subject.
