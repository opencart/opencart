Response

\Opencart\System\Library\Response collects the body, headers and compression settings the framework will emit at the end of the request. Controllers write into it; the bootstrap calls output() once everything is rendered.

API

MethodDescription
addHeader(string $header)Queue a header. Pass the full line (e.g. 'Content-Type: application/json').
getHeaders(): arrayCurrently queued headers.
setOutput(string $output)Replace the body buffer.
getOutput(): stringRead the current body buffer (used by the bootstrap and by controller/*/after listeners).
setCompression(int $level)Set zlib compression level (0-9). The library only compresses if the client advertised support via Accept-Encoding.
redirect(string $url, int $status = 302)Send a Location redirect and exit().
output()Flush headers and body. Called by the bootstrap; you do not call it yourself.

Returning HTML

```php $this->response->setOutput($this->load->view('common/dashboard', $data)); ```

Calling setOutput replaces the buffer entirely. To append, read first:

```php $this->response->setOutput($this->response->getOutput() . $extra_html); ```

Returning JSON

```php $json = ['success' => true, 'message' => $this->language->get('text_success')]; $this->response->addHeader('Content-Type: application/json'); $this->response->setOutput(json_encode($json)); ```

Always set the content type explicitly -- the default is text/html.

Redirects

```php $this->response->redirect($this->url->link('common/dashboard', 'user_token=' . $this->session->data['user_token'], true)); ```

The third argument to $this->url->link(...) forces HTTPS where the storefront supports both schemes. redirect() calls exit(), so anything after it is dead code.

Compression

Set once during bootstrap (system/framework.php reads config_compression and applies the level via setCompression()). Avoid toggling it from request code -- some routes stream binary (image, downloads) and double-compressing breaks them.
