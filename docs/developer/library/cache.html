Cache

\Opencart\System\Library\Cache is a thin facade over a pluggable adaptor. The active adaptor is selected by the cache_engine config key and constructed at request bootstrap.

Adaptors

EngineFileNotes
filesystem/library/cache/file.phpDefault. Stores entries in system/storage/cache/.
apcusystem/library/cache/apcu.phpPer-process shared memory; fast but local to one PHP-FPM pool.
mem / memcachedsystem/library/cache/mem(cached).phpNetwork cache; survives across PHP processes and hosts.
redissystem/library/cache/redis.phpRedis-backed.

API

MethodDescription
get(string $key): mixedReturns the cached value or false if missing/expired.
set(string $key, mixed $value, int $expire = 0): voidStores a value. $expire is seconds; 0 uses the engine's default (cache_expire from config, which is 3600 by default).
delete(string $key): voidRemoves a single entry.
clear(): voidWipes the entire cache for the active adaptor.

Usage

```php $key = 'product.' . (int)$product_id; $product = $this->cache->get($key); if (!$product) { $product = $this->model_catalog_product->getProduct($product_id); $this->cache->set($key, $product, 3600); } ```

Cache values must be serialisable. Avoid caching closures or open resources.

Naming

Use dotted prefixes (product., category.path., ...) so clear()-style cleanup can be reasoned about. The framework already follows this scheme for built-in caches.
