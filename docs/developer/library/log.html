Log

\Opencart\System\Library\Log is a tiny line-based file logger. Each write() call appends a timestamped row to a single file under system/storage/logs/.

API

MethodDescription
__construct(string $filename)The log file's basename. The library prepends the storage path. Empty filename means "do nothing" -- writes are silently dropped.
write(mixed $message): voidAppends one line. If $message is not a string the library print_r's it.

Built-in instance

The framework constructs one log instance at bootstrap with the filename from config('error_filename') and stores it on the registry as log. Controllers reach it through $this->log:

```php $this->log->write('User ' . $this->user->getUserName() . ' edited setting ' . $key); ```

Per-extension log

If your extension produces a lot of output you do not want polluting the main error log, instantiate a dedicated logger:

```php $log = new \Opencart\System\Library\Log('hello.log'); $log->write('--- run start ---'); ```

The file ends up at system/storage/logs/hello.log.

Where errors land

The error / exception handlers in system/framework.php and cron.php call $log->write(...) for unhandled errors when error_log is enabled. The Maintenance → Error Logs page in admin reads the same file.

Rotation

OpenCart does not rotate logs. Configure logrotate on the host or wire a cron that prunes old lines.
