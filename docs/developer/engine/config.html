Config

\Opencart\System\Engine\Config is the framework's settings store. It loads PHP config files (the ones that fill $_['key']) into a flat key/value map and exposes simple get / set / has methods.

API

MethodDescription
get(string $key): mixedReturns the value or the empty string when missing.
set(string $key, mixed $value): voidStores or replaces a value.
has(string $key): boolTrue if the key has been set.
load(string $filename): arrayLoads a config file and merges its $_ array into the store.
addPath(string $namespace, string $directory = ''): voidEither sets the base directory (one-arg form) or adds an override directory for a particular namespace prefix.

Loading config files

A config file is a plain PHP file that fills the $_ array, similar to a language file:

```php $config->load('admin') resolves the file to system/config/admin.php (the base directory was set with addPath() earlier), requires it, and merges the resulting $_ array into the internal data store. Existing keys are overwritten silently.

What gets loaded at request start

  1. system/config/default.php -- shared defaults (DB engine, session engine, error filename, etc.).
  2. system/config/admin.php or catalog.php -- pre-action and event maps for that side.
  3. Database settings via startup/setting -- merges oc_setting rows for the current store into the same map.
  4. Per-extension config keys when startup/extension activates each enabled extension.

By the time a controller runs, all four sources are blended into one flat map.

Add-paths and extension overrides

Two-argument addPath() registers a directory for a route namespace:

```php $config->addPath('extension/hello', DIR_EXTENSION . 'hello/system/config/'); ```

From then on, $config->load('extension/hello/module') resolves to that directory instead of the default. The startup chain uses this so each enabled extension can ship its own config files without colliding.

Using $this->config

```php if ($this->config->get('config_maintenance')) { // ... } $this->config->set('config_store_id', 0); if ($this->config->has('config_admin_language')) { $code = $this->config->get('config_admin_language'); } ```

Avoid mutating config from inside a request unless you genuinely need to (e.g. startup/setting does it to inject DB settings). Random $this->config->set(...) calls leak across the request and are hard to reason about.
