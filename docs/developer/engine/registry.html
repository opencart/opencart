Registry

The registry is OpenCart's service container. It is a typed map of string => object: every shared service the framework constructs (config, db, session, cache, language, ...) is stored on it, and every controller, model and library reads its dependencies from it via property access.

The five methods

MethodDescription
get(string $key): ?objectReturns the stored object or null if the key is unknown.
set(string $key, object $value): voidStores an object. Replaces any previous value silently.
has(string $key): boolTrue if the key is set.
unset(string $key): voidRemoves a key.

Values must be objects -- the registry rejects scalars at the type-hint level. For scalar configuration use Config, which is itself stored on the registry.

Magic property access

The registry implements __get, __set and __isset, so these are equivalent:

```php $registry->get('db'); $registry->db; $registry->set('cache', $cache); $registry->cache = $cache; $registry->has('session'); isset($registry->session); ```

Controllers, models and libraries also forward property access to the registry via the same magic methods on their base classes. That is why $this->db inside a controller resolves to $registry->get('db').

How the registry gets populated

The framework bootstrap (system/framework.php) constructs each service and stores it on the registry in a fixed order:

  1. autoloader -- PSR-4-style autoloader.
  2. config -- application config (default + admin/catalog).
  3. log -- error log writer.
  4. event -- event registry.
  5. factory -- builds controllers, models, libraries on demand.
  6. load -- the loader.
  7. request, response, db (if db_autostart), cache, template, language, url, document.

Pre-actions then add or replace entries (session, customer, currency, tax, user, etc.).

Using the registry from your code

You normally never touch the registry directly -- access services through $this. The exception is when you are writing a library and need to be registry-aware: extend the constructor with \Opencart\System\Engine\Registry $registry and the factory will hand it in.

```php namespace Opencart\System\Library\Hello; class Greeter { private \Opencart\System\Engine\Registry $registry; public function __construct(\Opencart\System\Engine\Registry $registry) { $this->registry = $registry; } public function hello(): string { $config = $this->registry->get('config'); return 'Hello from ' . $config->get('config_name'); } } ```
