URL

\Opencart\System\Library\Url turns route + query strings into absolute URLs. The catalog also chains a "rewrite" object onto it that converts the raw URL into an SEO-friendly form before output.

Constructor

```php $url = new \Opencart\System\Library\Url($base); ```

$base is the storefront's site URL (configured via HTTP_SERVER in config.php). Internally the library appends ?route=... and any extra args.

API

MethodDescription
link(string $route, string|array $args = '', bool $js = false): stringBuild a URL.
addRewrite(object $rewrite)Register a rewrite handler. Anything with a rewrite($url): string method qualifies; the catalog wires $this->seo_url here so SEO URLs work everywhere.

Building links

```php $href = $this->url->link('product/product', 'product_id=' . $product_id); $href = $this->url->link('common/dashboard', 'user_token=' . $this->session->data['user_token']); ```

Always pass the admin user_token (or the API session token) on links to authenticated pages. The router rejects requests that arrive without one.

Args

$args may be a query-string fragment ('foo=1&bar=2') or an associative array (['foo' => 1, 'bar' => 2]). The library encodes values with http_build_query when an array is supplied.

Js mode

Pass true as the third argument when the URL will be embedded in a JS string -- it returns a form safe to interpolate without further escaping. Most callers leave it false.

SEO URLs

The catalog startup chain instantiates the rewrite handler so https://store/index.php?route=product/product&product_id=42 rewrites to https://store/some-product on the way out. $this->url->link() is the only safe way to build a URL -- never concatenate ?route= by hand.
