Cron jobs

OpenCart treats scheduled jobs as plain controllers. Each entry in the oc_cron table points at a controller route; the dispatcher in catalog/controller/cron/cron.php walks that table and invokes any job whose cycle has elapsed.

The CLI entry point

upload/cron.php is the bootstrap script. It is conceptually similar to index.php but with three differences:

  • It registers the catalog namespace (cron jobs run as catalog controllers).
  • It does not run the action_pre_action chain. There is no session, no SEO URL parsing, no maintenance check.
  • It executes a single fixed action: cron/cron.

SAPI / secret guard

The first lines of cron.php reject HTTP traffic unless the caller proves it knows the CRON_SECRET env var:

```php if (PHP_SAPI !== 'cli') { $expected = (string)getenv('CRON_SECRET'); $provided = isset($_GET['secret']) ? (string)$_GET['secret'] : ''; if ($expected === '' || !hash_equals($expected, $provided)) { http_response_code(403); exit('Forbidden'); } } ```

This complements the Apache <Files cron.php> deny rule and matters on hosts that ignore .htaccess (nginx, IIS). If you want to allow web-triggered crons, set CRON_SECRET in your environment and call https://example.com/cron.php?secret=<value>; otherwise leave it unset and reach the script via CLI only.

The oc_cron table

Schema (from system/helper/db_schema.php):

ColumnTypeDescription
cron_idint auto-incrementPrimary key.
codevarchar(128)Internal identifier (unique per job).
descriptiontextFree-form, shown in admin.
cyclevarchar(12)One of hour, day, week, month, year. Read by strtotime('+1 <cycle>').
actiontextController route the dispatcher will load, e.g. cron/currency.
statustinyint(1)1 = enabled.
date_addeddatetimeWhen the row was created.
date_modifieddatetimeLast successful run -- used to compute the next due time.

Dispatcher

catalog/controller/cron/cron.php:

```php public function index(): void { $time = time(); $this->load->model('setting/cron'); $results = $this->model_setting_cron->getCrons(); $this->load->controller('cron/subscription', 3, 'subscription', 1, '', ''); foreach ($results as $result) { if ($result['status'] && (strtotime('+1 ' . $result['cycle'], strtotime($result['date_modified'])) < ($time + 10))) { $this->load->controller($result['action'], $result['cron_id'], $result['code'], $result['cycle'], $result['date_added'], $result['date_modified']); $this->model_setting_cron->editCron($result['cron_id']); } } } ```

The 10-second slack window absorbs scheduling jitter so a job set to run "every hour" still fires when the OS scheduler invokes cron.php a few seconds early. editCron() updates date_modified to the current time after the controller returns so the next due time is calculated from the most recent successful run.

The arguments passed to $this->load->controller() become the cron handler's index($cron_id, $code, $cycle, $date_added, $date_modified) parameters.

Built-in jobs

RouteCycleWhat it does
cron/currencydayRefreshes the currency exchange-rate table from the configured provider.
cron/gdprdayAnonymises GDPR-flagged customer/order rows after their retention window.
cron/subscriptiondayWalks oc_subscription and creates renewal orders for subscriptions whose next billing date has been reached. Always called by the dispatcher; not driven by an oc_cron row.

Admin UI

admin/controller/marketplace/cron.php exposes a list view at ?route=marketplace/cron with these actions:

  • Run now -- invokes the route directly through $this->load->controller(...) from the admin SAPI.
  • Enable / disable -- toggles the status column.
  • Edit -- changes the description (the route, code and cycle are owned by the extension that installed the row).
  • Delete -- removes the row. This is what an extension's uninstall method should do via $this->model_setting_cron->deleteCronByCode(...).

Writing your own cron job

Place a controller in your extension under extension/<your_module>/catalog/controller/cron/<name>.php:

```php namespace Opencart\Catalog\Controller\Extension\YourModule\Cron; class Cleanup extends \Opencart\System\Engine\Controller { public function index(int $cron_id, string $code, string $cycle, string $date_added, string $date_modified): void { $this->load->model('setting/setting'); // ... do work, e.g. delete stale tokens, recompute caches ... } } ```

Register the row in the install method of your extension:

```php $this->load->model('setting/cron'); $this->model_setting_cron->addCron( 'your_module_cleanup', 'Cleans up stale tokens', 'day', 'extension/your_module/cron/cleanup' ); ```

And remove it on uninstall:

```php $this->model_setting_cron->deleteCronByCode('your_module_cleanup'); ```

Scheduling on the host

Run cron.php as often as your shortest cycle requires. Hourly is enough if you do not have any "10 minutes" jobs (the framework does not support sub-hour cycles out of the box).

CLI (preferred):

``` 0 * * * * /usr/bin/php /path/to/opencart/upload/cron.php > /dev/null 2>&1 ```

HTTP fallback (only if you cannot use CLI):

``` 0 * * * * curl -s 'https://example.com/cron.php?secret=YOUR_CRON_SECRET' > /dev/null ```

Set CRON_SECRET in your web server's environment so the SAPI guard accepts the request. Treat the value like a password -- anyone with it can trigger every enabled cron.
