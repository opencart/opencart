Template

\Opencart\System\Library\Template is the rendering facade. It picks an adaptor at construction (currently always Twig) and exposes render().

Adaptors

EngineFileNotes
twigsystem/library/template/twig.phpDefault. Wraps the bundled Twig 3.
templatesystem/library/template/template.phpPlain PHP fallback. Not used by core templates.

API

MethodDescription
__construct(string $adaptor)Engine name.
render(string $filename, array $data = [], string $code = ''): stringRender a template. $filename is a route (no extension); $data becomes the variable scope; $code is reserved for caching.
addPath(string $namespace, string $directory = '')Register an extension's view directory so its templates resolve under a route prefix.

How $this->load->view() uses it

The loader fires view/<route>/before, then -- unless a listener filled $output -- calls $this->template->render($route, $data, $code), then fires view/<route>/after. You normally never call render() directly; you call $this->load->view(...).

Twig sandboxing

OpenCart does not enable Twig's sandbox by default. Templates can call any PHP function exposed via twig_safe filters or registered globals, so treat templates as code, not as configuration. Never load a template path that came from user input.

Caching

Twig compiles templates to PHP and caches them under system/storage/cache/template/. After editing a .twig file it is sometimes necessary to clear that directory (or hit Settings → Reset) to force recompilation on hosts that cache aggressively.
