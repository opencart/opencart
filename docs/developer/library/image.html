Image

\Opencart\System\Library\Image wraps GD-based image manipulation. Used internally by the catalog model for product thumbnails and on-the-fly resizes.

Constructor

```php $image = new \Opencart\System\Library\Image('/absolute/path/to/file.jpg'); ```

The constructor opens the file, reads dimensions / MIME / bits, and keeps the GD resource around for subsequent operations.

API

MethodDescription
getFile() / getImage()Path to the source file / GD resource.
getWidth() / getHeight()Pixel dimensions.
getBits() / getMime()Bits per channel and the detected MIME.
resize(int $width = 0, int $height = 0, string $default = '')Resize. $default controls the fit strategy ('w', 'h', '').
watermark(self $watermark, string $position = 'bottomright')Overlay another Image instance. Position is one of topleft, topright, bottomleft, bottomright.
crop(int $top_x, int $top_y, int $bottom_x, int $bottom_y)Crop to a rectangle.
rotate(int $degree, string $color = 'FFFFFF')Rotate by N degrees, filling exposed corners with $color (hex without #).
save(string $file, int $quality = 90)Encode and write to disk. The output format is inferred from the destination file extension.

Usage

```php $source = new \Opencart\System\Library\Image(DIR_IMAGE . 'catalog/' . $filename); $source->resize(400, 400); $source->save(DIR_IMAGE . 'cache/' . $cache_name); ```

For watermarks, both images must be opened first:

```php $image = new \Opencart\System\Library\Image(DIR_IMAGE . 'catalog/' . $filename); $mark = new \Opencart\System\Library\Image(DIR_IMAGE . 'logo.png'); $image->watermark($mark, 'bottomright'); $image->save(DIR_IMAGE . 'cache/' . $cache_name); ```

Model layer

For typical product images, do not instantiate Image directly -- use $this->model_tool_image->resize($filename, $width, $height). It handles cache invalidation and returns a public URL that points at the cached image.
