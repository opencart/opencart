Loader

\Opencart\System\Engine\Loader is the orchestrator. Every $this->load->X(...) call inside a controller or model goes through it. The loader resolves the route, fires the matching before event, builds the object via the factory, calls it, fires the matching after event, and returns the result.

controller()

```php $output = $this->load->controller('common/header'); ```

Resolves common/header → Opencart\Admin\Controller\Common\Header::index(), fires controller/common/header/before, calls the method with the trailing args, fires controller/common/header/after with the return value, and returns it.

If the method name is given via dot syntax (common/header.partial), that method is called instead of index. Methods starting with __ are blocked.

model()

```php $this->load->model('setting/setting'); $value = $this->model_setting_setting->getSetting('config'); ```

Loads the model class, then walks its public methods and writes a closure for each one onto a Proxy object. The proxy is exposed as $this->model_setting_setting. Every closure dispatches model/<route>/<method>/before, calls the real method, and dispatches model/<route>/<method>/after.

This proxy layer is what lets OCMOD redirect a single method to a patched copy without rewriting any caller.

view()

```php $html = $this->load->view('common/dashboard', $data); ```

Fires view/<route>/before with $route, $data, $code and an empty $output. If a listener fills $output the loader uses it directly (skips Twig). Otherwise $this->template->render(...) renders view/template/<route>.twig. view/<route>/after fires last and can rewrite the output.

language()

```php $this->load->language('common/dashboard'); $title = $this->language->get('heading_title'); ```

Fires language/<route>/before, calls Language::load($route, $prefix, $code), fires language/<route>/after. The returned map is also merged into $this->language so get() / all() calls see the new keys.

library()

```php $this->load->library('cart/cart'); $cart = $this->cart; ```

Asks the factory to instantiate Opencart\System\Library\Cart\Cart, registers it on the registry under cart (the last segment of the route), and returns it. The instance is cached -- a second call returns the same object. library/<route>/before and after fire around the build.

config()

```php $entries = $this->load->config('marketplace/extension'); ```

Loads a config file by route. Used mostly internally; most code talks to the global $this->config instead.

helper()

```php $this->load->helper('general'); ```

Requires system/helper/<route>.php once. Helpers are plain functions (not classes) so they do not go through the factory or fire events.

Route sanitisation

Each method strips characters that should never appear in a route. The exact alphabet differs slightly by method:

MethodAllowed in route
controllera-zA-Z0-9_/.| (the . separates method name)
model / librarya-zA-Z0-9_/
viewa-zA-Z0-9_/
language / configa-zA-Z0-9_-/ (allows en-gb-style codes)

This is a defence-in-depth check, not a validator -- never pass user input as a route.
